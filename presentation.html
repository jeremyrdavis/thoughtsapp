What Are Vector Embeddings?

  • Text is converted into a high-dimensional numeric vector (e.g. 768 floats)
  • Semantically similar text produces vectors that are close together in vector space
  • Generated by a neural network model (e.g. nomic-embed-text)

"I love helping people" → [0.023, -0.118, 0.457, ..., 0.091]
"You are worthless"     → [0.891, -0.034, -0.672, ..., 0.445]

These vectors capture meaning, not just keywords.

Cosine Similarity

Measures the angle between two vectors, ignoring magnitude.


cos(θ) = (A · B) / (||A|| × ||B||)


ScoreMeaning
1.0Identical direction — same meaning
0.85+Very similar — likely matches category
0.5Somewhat related
0.0Orthogonal — unrelated
-1.0Opposite direction — opposite meaning

Why Cosine Similarity?

Advantages

  • Ignores vector magnitude — focuses on direction (meaning)
  • Fast to compute, even for high-dimensional vectors
  • Works natively in PostgreSQL via pgvector (<=> operator)
  • No model fine-tuning required

In Our App

  • Compare incoming thought to 6 reference vectors
  • 3 positive + 3 negative reference phrases
  • If similarity to any negative reference > 0.85 → REJECTED
  • Otherwise → APPROVED

Cosine Similarity in pgvector

The <=> operator computes cosine distance (1 - similarity).

-- Find nearest negative vector and compute similarity
SELECT id, label,
       (1 - (embedding <=> cast(:vec AS vector))) AS similarity
FROM evaluation_vectors
WHERE vector_type = 'NEGATIVE'
ORDER BY embedding <=> cast(:vec AS vector) ASC
LIMIT 1;

This single query replaces what would otherwise require loading all vectors into application memory and computing similarity in Java.

Running the Embedding Model

The service uses Ollama to serve the embedding model locally.

# 1. Install Ollama
brew install ollama

# 2. Pull the embedding model
ollama pull nomic-embed-text

# 3. Start the server (default port 11434)
ollama serve

# 4. Verify
curl http://localhost:11434/v1/embeddings \
  -H "Content-Type: application/json" \
  -d '{"model": "nomic-embed-text", "input": "Hello world"}'

# 5. Initialize reference vectors
curl -X POST http://localhost:8088/vectors/initialize

Key Code: EvaluationServiceImpl

@Transactional
public ThoughtEvaluation evaluateThought(UUID thoughtId, String content) {
    // 1. Generate embedding vector from thought text
    float[] thoughtVector = embeddingService.generateEmbedding(content);

    // 2. Find nearest negative reference vector using pgvector
    List<Object[]> results = em.createNativeQuery(
        "SELECT id, label, " +
        "  (1 - (embedding <=> cast(:vec AS vector))) AS similarity " +
        "FROM evaluation_vectors WHERE vector_type = 'NEGATIVE' " +
        "ORDER BY embedding <=> cast(:vec AS vector) ASC LIMIT 1")
        .setParameter("vec", vectorLiteral)
        .getResultList();

    double maxSimilarity = ((Number) results.get(0)[2]).doubleValue();

    // 3. Compare against threshold
    ThoughtStatus status = maxSimilarity > similarityThreshold
        ? ThoughtStatus.REJECTED
        : ThoughtStatus.APPROVED;

    // 4. Persist and return result
    return createEvaluation(thoughtId, status, maxSimilarity, ...);
}

Key Takeaways

  • Microservices — Independently deployable Quarkus services communicating via Kafka
  • Event-driven — Thoughts are evaluated asynchronously, decoupled from the user request
  • AI integration — Vector embeddings + cosine similarity for semantic text evaluation
  • pgvector — Similarity search pushed down to the database for performance
  • Ollama — Run AI models locally with an OpenAI-compatible API
  • Cloud-native — Health checks, metrics, container images, OpenShift-ready
